Tick Queue System — OSRS-Authentic Scheduling

6 March 2026

OSRS doesn't run in real-time. It runs in ticks — discrete 600ms intervals where every game action is processed. This is the heartbeat of the game, and getting it right is non-negotiable for authenticity.

600ms — The Sacred Interval

Every 0.6 seconds, the OSRS server processes one game tick. During that tick, in a specific order:

  1. Player movement is processed
  2. Player actions are executed (combat, skilling, etc.)
  3. NPC actions are processed
  4. Projectiles advance
  5. Delayed effects resolve (hitsplats, XP drops)
  6. State is broadcast to clients

MiniScape replicates this with a setInterval running at exactly 600ms. Every scheduled action goes through the tick queue.

The Queue

Actions in OSRS don't happen "now" — they happen "on tick N." When you click to attack an NPC, the game schedules your first attack for the next available tick based on your weapon speed. When that attack hits, the hitsplat might be delayed further if it's a ranged or magic attack.

// Schedule a delayed action
function scheduleTick(player, ticksFromNow, callback) {
  const targetTick = currentTick + ticksFromNow;
  tickQueue.push({ player, tick: targetTick, fn: callback });
}

// Every 600ms
setInterval(() => {
  currentTick++;

  // Process movement
  players.forEach(processMovement);

  // Process tick queue
  tickQueue
    .filter(t => t.tick === currentTick)
    .forEach(t => t.fn());

  // Broadcast state
  broadcastGameState();
}, 600);

Why Ticks Matter

If you've ever played OSRS, you've experienced tick manipulation — prayer flicking, 1-tick cooking, 3-tick fishing. These techniques exploit the tick system's deterministic timing. If MiniScape's tick system doesn't match OSRS exactly, none of these techniques would work.

Some examples of tick-dependent mechanics:

  • Attack speed — A bronze sword attacks every 4 ticks (2.4s). Not 2.3s, not 2.5s.
  • Eating delay — Eating food delays your next attack by 3 ticks.
  • Run energy — Regenerates 1% per tick while standing still at level 1 Agility.
  • Respawn timers — NPCs respawn on a fixed tick count after death.
  • Projectile travel — Arrows travel at a fixed rate per tick based on distance.
The tick is the atom of OSRS. Everything is built on it, and everything must respect it.
