Lumbridge & Terrain Rendering from OSRS Cache

5 March 2026

The OSRS game world is massive — thousands of regions, each containing 64x64 tiles of terrain data, object placements, and collision flags. MiniScape renders this world in the browser using data extracted from the game cache.

Cache Structure

The OSRS cache stores map data in two file types per region:

  • Map files (m{x}_{y}.dat) — terrain heights, overlays, underlays, tile flags
  • Location files (l{x}_{y}.dat) — object placements (trees, rocks, buildings, doors)

Each region is identified by coordinates. Lumbridge sits at region 12850 (x=50, y=50 in region coords), which maps to world tiles 3200-3263, 3200-3263.

Tile Rendering

Each tile has several properties that affect rendering:

  • Underlay — the base terrain color (grass, dirt, sand, water)
  • Overlay — an optional second layer (roads, floors, special terrain)
  • Height — vertical offset for 3D-like terrain variation
  • Flags — collision, bridge, roof visibility

MiniScape uses a top-down 2D renderer. Each tile is drawn as a colored rectangle, with color determined by the underlay/overlay definitions. Objects like trees and rocks are drawn as sprites on top.

Collision Maps

Every tile has collision flags that determine what can move through it. A tile might be:

  • Fully walkable
  • Blocked on specific edges (north wall, east wall, etc.)
  • Completely blocked (solid object)
  • Projectile-blocking (affects ranged/magic line of sight)

MiniScape's pathfinding respects these flags exactly — if you can't walk somewhere in OSRS, you can't walk there in MiniScape.

The world isn't designed — it's extracted. Every tile, every collision flag, every object placement comes directly from the game cache.
