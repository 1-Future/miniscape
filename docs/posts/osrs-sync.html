OSRS Sync — Real-Time Data Mirroring

7 March 2026

Most private servers guess at game values or copy them from wikis. MiniScape takes a different approach: we capture data directly from the live game and mirror it into our server. No interpretation, no estimation — raw ground truth.

The Pipeline

The data capture pipeline has three stages:

  1. Capture — A RuneLite plugin intercepts game events in real-time
  2. Log — Events are written to ~/.runelite/combat-debug.log with timestamps
  3. Extract — We parse the log and encode values into MiniScape's data files

What We Capture

The Combat Debug Plugin hooks into RuneLite's event system to capture:

  • NPC definitions — ID, name, combat level, position, animations
  • Combat events — attack animations, hitsplats, damage values, projectile IDs
  • Inventory changes — item IDs, quantities, slot positions
  • Skill XP — exact XP gains per action
  • Player stats — levels, equipment bonuses, attack styles
  • Animations — IDs for every action (fishing, woodcutting, cooking, etc.)
  • Coordinates — exact tile positions for NPCs, objects, and the player

Log Format

Each log entry is timestamped and categorized:

[14:32:05] === SESSION START ===
[14:32:08] Skills: Attack=1 Strength=1 Defence=1 ...
[14:33:15] NPC interaction: Gielinor Guide (id=3308) at (3095, 3106)
[14:34:22] Inventory: +1 Small fishing net (303)
[14:35:01] Animation: 621 (fishing)
[14:35:04] XP: Fishing +10 (total: 10)
[14:36:12] Combat: hit 1 on Giant rat (anim: 386)

Why Not Just Read the Wiki?

The OSRS Wiki is excellent, but it has limitations:

  • Animation IDs aren't documented for most actions
  • Exact XP values for tutorial-specific actions aren't always listed
  • NPC positions on Tutorial Island differ from their post-tutorial versions
  • Projectile travel times and delay formulas need empirical verification
  • The wiki documents what happens — we need to know exactly when it happens, down to the tick
The log file is the single source of truth. If MiniScape behaves differently from what the log says, MiniScape is wrong.
