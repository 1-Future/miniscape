Combat

OSRS Combat Styles & Projectile Delays

7 March 2026

Combat in OSRS is deceptively complex. What looks like "click and wait" is actually a precisely timed system of attack speeds, projectile travel, hitsplat delays, and XP distribution. We captured it all.

The Three Combat Styles

Melee

Melee is instant — damage is calculated and applied on the same tick the animation plays. A bronze sword swings at 4-tick speed (2.4 seconds) with animation ID 386.

XP is distributed at 4 XP per damage to the selected style (Attack, Strength, or Defence) plus 1.33 XP per damage to Hitpoints.

Ranged

Ranged introduces projectile travel time. The arrow fires on tick 0 (animation 426), but the hitsplat doesn't appear until the projectile arrives. Travel time depends on distance:

DistanceTravel TicksTotal Delay
1–3 tiles10.6s
4–6 tiles21.2s
7–10 tiles31.8s

Shortbow attack speed is 4 ticks. XP: 4 Ranged XP per damage + Hitpoints.

Magic

Magic works like ranged with projectile delay, but adds a spell lookup layer. Wind Strike (anim 711) has a max hit of 2 and grants 5.5 base XP plus 2 XP per damage to Magic, plus Hitpoints XP.

Every weapon has an attack speed measured in game ticks (0.6s each):

Damage Calculation

OSRS uses a two-roll system: first an accuracy roll to determine if the attack hits, then a damage roll from 0 to max hit. Both rolls are uniform random. We replicate this exactly — no simplifications, no approximations.

// Accuracy: attack roll vs defence roll
const attackRoll = Math.floor(Math.random() * (effectiveLevel * (equipBonus + 64)));
const defenceRoll = Math.floor(Math.random() * (defLevel * (defBonus + 64)));
const hit = attackRoll > defenceRoll;

// Damage: uniform 0 to maxHit (inclusive)
const damage = hit ? Math.floor(Math.random() * (maxHit + 1)) : 0;
When the Combat Debug Plugin logs "hit 1" on a giant rat, MiniScape's combat system produces statistically identical results. Same formulas, same distributions.
